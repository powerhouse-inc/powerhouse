# Powerhouse v6.0.0 🚀

## ✨ Highlights

1. **Redesigned Reactor with sync reliability** - Complete rewrite of the write model, sync pipeline, and job system
2. **Vetra Package Ecosystem** - Private npm-compatible registry with `ph publish`, `ph install`, `ph unpublish`, and runtime package loading
3. **Reactor Attachments** - New package for document-linked file storage
4. **Switchboard Load Balancer** - Built-in `switchboard-lb` for multi-node deployments
5. **Full Observability** - OpenTelemetry, Sentry source maps, Prometheus exporter

This is the **first production release of the 6.x line**, replacing 5.3.6. It is a breaking major release — existing Powerhouse projects must run `ph migrate 6.0.0` to upgrade.

## ⚠️ Upgrading from 5.x

The canonical way to upgrade an existing Powerhouse project is a single command:

| Command                | Description                                                                |
| ---------------------- | -------------------------------------------------------------------------- |
| `ph migrate 6.0.0`     | Migrate to a specific version (recommended)                                |
| `ph migrate latest`    | Migrate to whatever npm currently tags `latest`                            |
| `ph migrate --force`   | Skip the codegen-version match check (registry unreachable / local build)  |
| `ph migrate --debug`   | Print resolution decisions and the re-execution command                    |

This is **not just a dependency bump**. `ph migrate` runs codegen against the new layout and applies every structural change required by 6.x:

- **Restructures document models** to the versioned `v1/` layout
- **Updates `package.json`** — exports, scripts, peer/dev dependencies; workspace pins to the target version
- **Overwrites root config files** (tsconfig, eslint, vitest, build configs)
- **Rewrites legacy import paths**
- **Deletes `src/subgraphs/`** (now generated at runtime by Switchboard)
- **Reinstalls dependencies** with your detected package manager
- **Re-runs codegen** to regenerate everything

Before running, confirm Node 24:

```bash
node --version   # must be >= 24.0.0
```

If Node is older: `nvm install 24 && nvm use 24` first.

## ⚙️ Redesigned Reactor

The reactor's core write model was rearchitected to unify transactions with execution context. The sync pipeline gained per-document quarantine (instead of blocking entire remotes), a persistent dead-letter store, buffered batched mailboxes, abort signals on GQL channels, and cursor-based paging for large sync batches.

Key additions:

- **DocumentIntegrityService** - validates and rebuilds document keyframes/snapshots
- **Sync paging** - `getOperations` now pages across scopes
- **Job timeout** via abort signals propagated through the entire job pipeline
- **Exponential backoff + jitter** on retries
- **`test-sync-queue` CLI app** for detecting sync drift on large drives

## 📦 Vetra Package Ecosystem

A full end-to-end package ecosystem now powers Powerhouse apps. A private npm-compatible registry (Verdaccio-backed) hosts packages, and the CLI, Connect, and Switchboard all integrate with it.

| Command                                       | Description                                              |
| --------------------------------------------- | -------------------------------------------------------- |
| `ph publish`                                  | Publish a built package to the Powerhouse registry       |
| `ph install @your-org/your-package`           | Install from the registry (Powerhouse registry default)  |
| `ph unpublish @your-org/your-package@1.0.0`   | Unpublish and purge the CDN cache                        |

**Dynamic loading:** Connect and Switchboard load processors and subgraphs at runtime from the HTTP registry — no rebuild required when new packages are published.

**Auto-discovery:** Connect automatically finds and installs the correct package when it encounters an unknown document type.

**Version picker:** The Connect UI lets users choose specific dist-tag versions when installing packages.

## 📎 Reactor Attachments

New `reactor-attachments` package provides document-linked file storage, integrated directly into the reactor and switchboard.

- Upload reservations + direct upload to storage
- Switchboard transport implementation
- `HEAD` support, soft-delete, and case-insensitive hash indexing
- `Attachment-Metadata` headers

## ⚖️ Switchboard Load Balancer

New `switchboard-lb` package provides HTTP load balancing for multi-node Switchboard deployments:

- Least-connections proxying of upstream routes
- Request pinning across restarts
- Rewrite to drive-id header for deterministic routing

## 📊 Observability

OpenTelemetry, Sentry, and Prometheus are first-class citizens in 6.x.

```bash
# Enable OTel metrics export from Switchboard
OTEL_EXPORTER_OTLP_ENDPOINT=http://tempo:4318 ph switchboard start

# Prometheus metrics (opt-in)
PROMETHEUS_METRICS=true ph switchboard start
```

- New `opentelemetry-instrumentation-reactor` package for distributed tracing across reactor internals
- Sentry source maps uploaded in CI and releases tagged automatically
- OTel spans bridged to Sentry for unified error correlation
- Opt-out error reporting for `ph-cli` and `ph-cmd` via config

## 🪝 `useDocumentSafe` Hook

Replaces the pattern of calling `useDocument` and catching thrown errors — error and loading state are now first-class values.

```typescript
import { useDocumentSafe } from "@powerhousedao/reactor-browser";

function MyEditor({ documentId }: { documentId: string }) {
  const { document, error, isLoading } = useDocumentSafe(documentId);
  if (isLoading) return <Spinner />;
  if (error) return <ErrorBanner error={error} />;
  return <Editor document={document} />;
}
```

## ⚠️ Breaking Changes

> Most of these are handled automatically by `ph migrate 6.0.0`. If you upgrade by hand, address each one explicitly.

| Change                                  | Handled by `ph migrate`? |
| --------------------------------------- | ------------------------ |
| Node.js 24 minimum                      | ❌ Manual (`nvm install 24 && nvm use 24`) |
| Versioned document model layout (`v1/`) | ✅ Automatic              |
| Generated subgraphs removed             | ✅ Automatic              |
| `DriveEditor` rename                    | ❌ Manual (follow TypeScript guidance) |
| `package.json` shape changed            | ✅ Automatic              |
