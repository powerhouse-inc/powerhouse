## Gap Report: Reference — Authorization (Renown SDK)

Reviewed: docs/academy/04-Reference/06-Authorization (00-Overview.md, 01-Authentication.md, 02-APIReference.md, 03-CLIIdentity.md)
Against: packages/renown (`@renown/sdk`)
Focus: Renown SDK method names, authentication flow steps, hook and component exports, CLI identity commands

> Note: `packages/renown` (`@renown/sdk`) exports a **builder-based SDK** (`RenownBuilder`, `BaseRenownBuilder`, `Renown`, `ConnectCrypto`, etc.). The doc set documents an entirely different **React provider/hook/component API** (`RenownUserProvider`, `RenownAuthButton`, `useUser`, plus free functions `login`/`logout`/`openRenown`/etc. and a `SessionStorageManager` class). None of those documented names exist in the `@renown/sdk` barrel (`src/index.ts`). The React components/hooks that do exist (`RenownAuthButton`, `RenownLoginButton`, `useUser`) live in `packages/reactor-browser/src/renown/`, NOT in `@renown/sdk` — so even where a name matches, the documented import path `@renown/sdk` is wrong. This is the root cause of nearly every finding below.

The flat `name/email/avatar/ethAddress` fields could in principle originate there; finding #17 is anchored to what renown itself defines (`profile`, `ens`).
- Switchboard exports `getConnectCrypto`, `getConnectDid`, `getBearerToken` from `@powerhousedao/switchboard` (03-CLIIdentity.md:177-194, 307-320): the `apps/switchboard` package is outside the `packages/renown` review scope and was not loaded.
- `ph login --renown-url` default `https://renown.powerhouse.io` and `--timeout` default `300` (03-CLIIdentity.md:289-295): the `loginArgs` definition lives in `@powerhousedao/shared/clis/args`, outside scope; could not confirm the default URL string (note: `@renown/sdk`'s own `DEFAULT_RENOWN_URL` is `https://www.renown.id`, which differs).
- `ph switchboard --use-identity / --keypair-path / --require-identity` flags (03-CLIIdentity.md:138-163, 297-303): defined in `clis/ph-cli`, outside the `packages/renown` scope.
- Runtime behaviors (sessionStorage persistence, browser redirect handling, `window.renown`/`window.connectCrypto` globals) — require runtime verification and/or the reactor-browser package, out of scope.

### Summary
21 findings (16 stale, 0 missing, 5 wrong — items #18/#19 counted as wrong, #20 counted as stale). The Authorization reference docs describe a React `@renown/sdk` (provider + `useUser` + auth-button components + a flat set of free functions and a `SessionStorageManager`) that does not exist in the `packages/renown` source; the real `@renown/sdk` is a headless builder-based SDK, and the React surface actually lives in `packages/reactor-browser`. A developer following almost any API-reference or quick-start import would fail to compile. The CLI identity guide is closer to reality but cites the wrong credential file names (`.keypair.json`/`.auth.json` vs `.ph/.renown.json`). This section needs a substantial rewrite against the actual builder API and a corrected import-path story.
