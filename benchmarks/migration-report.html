What changed

Files added

  • tsconfig.tsgo.json — references list for tsgo (omits apps/academy)
  • .oxlintrc.json — oxlint config (type-aware, custom plugins, react-hooks v7 via jsPlugins)
  • .oxfmtrc.json — migrated from .prettierrc.json
  • eslint-plugins/{logger,cli-cold-path}.js — custom rules as standalone ESLint v9 plugin modules
  • benchmarks/{run.sh, diff.mjs, results.json}

Scripts added

  • pnpm typecheck:tsgo
  • pnpm lint:ox / pnpm lint:ox:fix
  • pnpm format / pnpm format:check (oxfmt)

CI

tsgo runs as advisory pre-check. oxlint is now the primary lint gate; oxfmt --check gates formatting; slim ESLint runs only eslint-plugin-better-tailwindcss.

ESLint — vestigial only

  • No rules enabled. Plugins registered (@typescript-eslint, react-hooks, logger) so existing inline eslint-disable comments resolve. Delete once source files no longer reference those rule names in disable comments.

Moved to oxlint

  • typescript-eslint type-aware rules (via oxlint-tsgolint)
  • react-hooks/* (core) + react-hooks v7 compiler rules via jsPlugins (set-state-in-effect, incompatible-library, static-components)
  • eslint core (incl. no-useless-assignment nursery), unicorn/*, import/*, promise/*
  • Custom logger/* + cli-cold-path/* (loaded via jsPlugins)
  • tailwindcss/* via native oxlint-tailwindcss — 23 rules including no-unknown-classes, no-conflicting-classes, enforce-canonical, enforce-sort-order. Requires build:css first.

Moved to oxfmt

  • Formatting (replaces prettier). oxfmt --check is chained into lint:ox so formatting drift fails the lint script with non-zero exit.

Breaking changes & caveats

oxfmt (replaces prettier)

IssueImpactStatus
Markdown / MDX oxfmt corrupts MDX JSX-comment syntax ({/* ... */} → {/_ ... _/}), breaking academy docs and other generated MD files. handled **/*.md and **/*.mdx in .oxfmtrc ignore list. Markdown formatting still runs via prettier if desired (not gated by CI).
Output divergence from prettier Found one stylistic difference: oxfmt parenthesizes infer T in conditional types where prettier does not. No other differences observed across 2270 files. accepted
Editor integration VSCode "Format on Save" via the prettier extension still works if prettier remains in devDependencies; for native oxfmt experience, install oxc.oxc-vscode and set the formatter to oxfmt. informational

oxlint-tailwindcss (native plugin) — adopted

IssueImpactStatus
pnpm exports field resolution pre-build The native plugin loads each workspace's design system by following @import chains through package.json#exports. This repo's design-system / vetra / connect / powerhouse-vetra-packages map ./style.css → ./dist/style.css, which doesn't exist until build:css runs. Without it the plugin falls into a slow retry path: 38s on apps/connect/src alone, ~84s on full repo. resolved pnpm lint:ox now runs pnpm build:css first (~4s, builds the four Tailwind dist files in parallel). Full lint+format then takes ~28s on full repo, with the native plugin enabled. Also added array-fallback exports ("./style.css": ["./dist/style.css", "./style.css"]) as a hint, though Node's exports resolution does not actually honor array fallback for missing files — the build:css step is the real fix.
Stricter than better-tailwindcss Native plugin surfaces 58 unknown-class errors vs ~44 from the previous setup, plus enforce-canonical / sort-order / variant-order warnings that better-tailwindcss did not emit. triage
Tailwind v3 support oxlint-tailwindcss targets Tailwind v4 exclusively. This repo uses v4 so not an issue here. not blocking

tsgo (TypeScript 7 native preview)

Issue Impact Status
baseUrl compiler option fully removed in TS 7 apps/academy extends @docusaurus/tsconfig which still sets baseUrl: ".". ignoreDeprecations: "6.0" does not silence this in tsgo. workaround Excluded from tsconfig.tsgo.json. Long-term: wait for @docusaurus/tsconfig update or shim a local override.
Preview release (7.0.0-dev) API surface still moving; some tsc flags / edge cases may not be honored. advisory only tsc remains authoritative.
Editor support VSCode integration is via @typescript/native-preview extension, not the bundled TS Server. informational
Build cache tsgo writes its own .tsbuild/tsconfig.tsbuildinfo alongside tsc's — different format, no interop. handled Already in .gitignore patterns.

oxlint

Issue Impact Status
Custom plugin API Loads via jsPlugins in .oxlintrc.json. The repo's logger/* and cli-cold-path/* rules — already standard ESLint v9 plugins — run unchanged. handled Alpha; not semver-stable yet.
Type-aware rules 59 of 61 typescript-eslint type-aware rules supported via oxlint-tsgolint (Go). Runs once over the TS program, returns diagnostics into oxlint. handled Requires TypeScript 7+.
eslint-plugin-better-tailwindcss Could load via jsPlugins (fix landed Oct 2025), but the native oxlint-tailwindcss plugin can't resolve pre-build CSS here — see Tailwind caveats above. Stays in slim ESLint for now. eslint owns
Formatting oxlint does not format. Replaced eslint-plugin-prettier with oxfmt directly — no more lint/format wrapper. handled by oxfmt
no-useless-assignment Available in oxlint eslint plugin, nursery category. Enabled as warn. Same 8 findings as ESLint. migrated
react-hooks v7 compiler rules (set-state-in-effect, incompatible-library, static-components) oxc maintainers explicitly chose not to implement these natively (depend on React Compiler integration). Loaded instead via jsPlugins using the official eslint-plugin-react-hooks package, aliased to react-hooks-extra to avoid collision with oxlint's built-in react-hooks. migrated via jsPlugins
preserve-caught-error New ESLint 10 core rule. No oxlint equivalent and no tracking issue. 5 findings on this repo. Could be loaded via jsPlugins against @eslint/js. accepted gap
tsgolint memory on large repos Type-aware pass builds a TS program in Go; very large codebases may see high RSS. This monorepo (~2268 files) ran in 3s with no issue. no issue here
settings.react.version: "detect" not accepted Oxlint config schema expects a literal version string or omitted. handled Setting removed.

Recommended rollout

  1. Now (this PR) — tsgo as advisory pre-check, oxlint + oxfmt as blocking. Slim ESLint runs only better-tailwindcss.
  2. Next sprint — triage the 81 oxlint correctness errors (mostly real bugs: react-hooks/rules-of-hooks, no-unsafe-optional-chaining, unicorn/*), then flip oxlint's nursery-tier rules from warn to error.
  3. Done — native Tailwind plugin in use. oxlint-tailwindcss is loaded via jsPlugins; pnpm lint:ox runs build:css first so the compiled dist/style.css files exist for the plugin to resolve.
  4. 2–3 months — when tsgo hits 1.0 and @docusaurus/tsconfig drops baseUrl, swap pnpm typecheck to tsgo and keep tsc only as the CI authoritative gate.
