What changed

Files added

  • tsconfig.tsgo.json — references list for tsgo (omits apps/academy)
  • .oxlintrc.json — oxlint config tuned for this codebase
  • benchmarks/run.sh + results.json

Scripts added

  • pnpm typecheck:tsgo
  • pnpm lint:ox / pnpm lint:ox:fix

CI

Both new tools run as advisory pre-checks in .github/workflows/check-commit.yml with continue-on-error: true. They give sub-second feedback; the authoritative tsc/eslint passes still gate merges.

Kept as-is

  • tsc --build — authoritative typecheck
  • eslint — type-aware rules, prettier formatting, custom plugins (logger/*, cli-cold-path/*), better-tailwindcss
  • prettier — formatting (oxlint does not format)

This is intentionally a hybrid migration. The Rust/Go tools cover the fast path; the JS originals remain authoritative for anything they don't yet support.

Breaking changes & caveats

tsgo (TypeScript 7 native preview)

IssueImpactStatus
baseUrl compiler option fully removed in TS 7 apps/academy extends @docusaurus/tsconfig which still sets baseUrl: ".". ignoreDeprecations: "6.0" does not silence this in tsgo. workaround Excluded from tsconfig.tsgo.json. Long-term: wait for @docusaurus/tsconfig update or shim a local override.
Preview release (7.0.0-dev) API surface still moving; some tsc flags / edge cases may not be honored. advisory only tsc remains authoritative.
Editor support VSCode integration is via @typescript/native-preview extension, not the bundled TS Server. informational
Build cache tsgo writes its own .tsbuild/tsconfig.tsbuildinfo alongside tsc's — different format, no interop. handled Already in .gitignore patterns.

oxlint

IssueImpactStatus
No plugin API (yet) The repo's custom rules — logger/missing-token-args, logger/extra-args-without-token, cli-cold-path/allowed-static-imports — cannot run under oxlint. unsupported ESLint remains authoritative for these.
No type-aware rules Most typescript-eslint/* rules used here are type-aware: no-floating-promises, no-misused-promises, require-await, no-unnecessary-condition, restrict-template-expressions, consistent-type-imports, etc. Oxlint cannot evaluate them. unsupported ESLint remains authoritative.
No eslint-plugin-better-tailwindcss Tailwind class validation and ordering rules for design-system / connect / vetra packages. unsupported ESLint keeps these.
No eslint-plugin-prettier Oxlint does not format. Prettier still runs via ESLint, husky, and editor save. expected
settings.react.version: "detect" not accepted Oxlint config schema expects a literal version string or omitted. handled Setting removed.
Rule-name divergence Oxlint groups rules by plugin (e.g. typescript/no-unused-vars, react-hooks/exhaustive-deps). ESLint config IDs do not map 1:1. migration cost Documented in .oxlintrc.json.
Pre-existing findings surfaced 66 correctness errors + 171 warnings on first run — rules ESLint does not have or has disabled. triage See breakdown below.

Recommended rollout

  1. Now (this PR) — tsgo + oxlint installed, scripts added, CI runs them as advisory. Zero blast radius.
  2. Next sprint — fix the 66 oxlint correctness errors, then flip oxlint to blocking in CI.
  3. 2–3 months — when tsgo hits 1.0 and @docusaurus/tsconfig drops baseUrl, swap pnpm typecheck to tsgo and keep tsc only for the CI authoritative gate.
  4. Later — evaluate Biome or dprint to replace prettier for formatting once oxlint covers the lint cases.
